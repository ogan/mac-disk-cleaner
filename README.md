# Mac Cleaner App - Documentation

## Project Purpose
This project is a desktop application designed to manage disk space and free up storage by cleaning unnecessary files on macOS devices.

## Features
The application scans and cleans files in the following categories:
1. **System Caches:** Cache files located in `~/Library/Caches`.
2. **System Logs:** System logs located in `~/Library/Logs`.
3. **Trash Bin:** Files in the trash (`~/.Trash`).
4. **Media Analysis Data:** Media analysis data generated by macOS in the background (`~/Library/Containers/com.apple.mediaanalysisd/Data`).
5. **App Leftovers:** Leftover folders from uninstalled applications (searches within `~/Library/Application Support`. Safe keywords and currently installed applications are excluded).

**Extra Features:**
- **Modern Interface:** A modern, dark/light theme compatible user interface (UI) developed with the `customtkinter` library.
- **Asynchronous Operations:** Scanning and cleaning operations are performed in the background using `threading`, so the interface does not freeze.
- **Size Formatting:** The sizes of the found unnecessary files are presented to the user in a readable format (KB, MB, GB).

## File Structure
- `app.py`: The main entry point of the application. It houses the `MacCleanerApp` class and manages the graphical user interface (GUI).
- `scanner.py`: Contains the business logic (`Scanner` class) related to scanning the system and deleting files. Helper functions such as size calculation and formatting are also here.
- `requirements.txt`: Lists the dependencies of the project (`customtkinter`, `psutil`).
- `run.sh`: A practical bash script used to activate the virtual environment (`venv`) and start the application.
- `MacCleaner.spec`: `PyInstaller` configuration file used to compile the application into a single executable `.app` or file.
- `venv/`: Python virtual environment folder.
- `build/` & `dist/`: Build outputs generated by PyInstaller.

## How to Run?
To run the application, you can use the following command in the terminal while in the project directory:
```bash
./run.sh
```

## Updates
- **Interface (UI) Improvements (May 14, 2026):** The visibility of the "CLEAN / DELETE" button has been increased by making it constantly active and moving it to the top right panel (next to the "Scan Now" button) to make the deletion process easily noticeable. To prevent the overflow/hiding issue, the initial size of the application has been increased to 700x700, and the font size of the directory texts on the main screen has been reduced (from 14 to 11).
- **Security and Correctness Fixes (May 14, 2026):** Serious security fixes have been implemented to make the source code suitable for sharing on platforms like Github:
  - **TOCTOU and Symlink Protection:** The use of `shutil.rmtree` has been replaced with a custom `safe_delete` function to prevent the risk of following symbolic links and deleting critical files elsewhere in the system (Symlink escape). Before folders and files are deleted, it is strictly verified whether they are within the targeted safe directories (e.g., `~/Library/Application Support`) using `os.path.realpath` and `os.path.commonpath`. Additionally, `os.path.islink` is used to delete the symlinks themselves, not their contents.
  - **Precise `find_leftovers` Matching:** The old loose substring logic has been removed. Now, the `Info.plist` files of applications under `/Applications` are read with `plistlib` to extract `CFBundleIdentifier` (e.g., `com.google.Chrome`) values. Application leftovers are identified securely using these official Bundle IDs and exact application names instead of just "a word containing x".
  - **Strict `safe_keywords` Check:** The error of considering folders like Pineapple Studio safe (or falsely deleting them) just because they contain the word "apple" has been fixed. Instead, exact match and prefix (`com.apple.`) match logic has been adopted.
  - **Prevented Error Swallowing:** Blocks that silently swallow errors like `try/except Exception: pass` have been fixed. Possible permission errors and broken folder states are detected and transferred to the UI/error list.
  - **Deletion Size Accuracy (Race Condition):** Situations that could occur when trying to read the size right before deleting a file and then deleting it have been prevented. Now, the freed space is calculated with 100% accuracy by getting the size of each individual file instantly as it is deleted using `os.walk` in the tree structure.
  - **Cache Folder Protection:** In response to the fact that wholesale deletion of the `~/Library/Caches` directory could break some developer/media applications, this option is set to off by default in the interface, and a warning text has been added.
